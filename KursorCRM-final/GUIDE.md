## 1. Первый запуск

```bash
# 1. Установка зависимостей
npm install

# 2. (рекомендуется) задать секрет
echo "JWT_SECRET=$(openssl rand -hex 32)" > .env

# 3. Запуск сервера
npm start
```

В консоли вы увидите:

```
🚀 KURSOR работает: http://localhost:3000
   API:   http://localhost:3000/api/health
   WS:    ws://localhost:3000/ws
```

Откройте `http://localhost:3000` в браузере. По умолчанию создастся администратор `admin / admin`.

## 3. Вход и роли

Откройте `http://localhost:3000` → введите логин и пароль.

| Роль | Куда перенаправляет |

## 4. Админ-панель

URL: `http://localhost:3000/admin/`

Меню сгруппировано по блокам:

- **Образование**: модули, задачи, материалы курсов, доступ преподавателей к курсам.
- **CRM**: филиалы, тарифы, карточки клиентов.
- **Учебный процесс**: группы, журнал занятий, домашние задания, календарь.
- **Люди**: пользователи (создание, роли, привязка родителя к ребёнку), права преподавателей.
- **Обзор**: сводка, уведомления.

### 4.1 Филиалы

`Админка → CRM → Филиалы`

**Создать филиал:**
1. Нажмите «Добавить филиал».
2. Введите название (например, `KURSOR — Ташенова`) и адрес (например, `ул. Ташенова, 8`).
3. Сохраните.

**Изменить / удалить:** клик по строке филиала → редактирование / кнопка «Удалить».

> Тестовая база содержит 3 филиала: **KURSOR — Ташенова**, **KURSOR — Жошы Хан**, **KURSOR — Сарыарка**.

### 4.2 Тарифы

`Админка → CRM → Тарифы`

**Поля тарифа:**
- **Название** (например, «Стандарт (8 занятий)»)
- **Количество занятий** (`visitsCount`)
- **Срок действия в днях** (`durationDays`, обычно 30)
- **Цена в тенге** (`price`)
- **Комментарий** — отображается клиенту
- **Отдельная оплата доп. занятий** (`extraLessonsSeparate`)

При выдаче тарифа клиенту в CRM-карточке остаток занятий (`visitsLeft`) автоматически пересчитывается = `visitsCount` тарифа.

### 4.3 Пользователи

`Админка → Люди → Пользователи`

**Создать пользователя:**
1. «Добавить пользователя» → выберите роль:
   - `admin` — администратор
   - `teacher` — учитель
   - `assistant` — ассистент учителя (права настраиваются отдельно)
   - `student` — ученик
   - `parent` — родитель
2. Заполните: имя, логин, пароль, возраст, языки, привязку к учителю.
3. Сохраните.

**Привязка родителя к ребёнку:**
1. Откройте профиль пользователя с ролью `parent`.
2. Найдите блок «Дети» → выберите учеников из списка.
3. Сохраните.

> На уровне API: `PUT /api/users/:parentId/children { children: [studentId, ...] }`

**Аватарка:**
- Меняется только владельцем аккаунта (страница профиля).
- PNG/JPG, до 2 МБ, проверка по magic bytes.

### 4.4 Группы

`Админка → Учебный процесс → Группы`

**Поля группы:**
- **Название** (например, «Python — Старт (10–12)»)
- **Курс** (`courseId`) — связанный модуль из контента
- **Филиал** (`branchId`)
- **Учитель** (`teacherId`) — необязательно
- **Ассистент** (`assistantId`) — необязательно
- **Тип занятия**: `main` (основное) / `extra` (дополнительное)
- **Статус**: `active` / `archived`

### 4.5 Расписание группы

Внутри карточки группы → вкладка «Расписание».

**Добавить слот:**
- День недели: `0=Вс`, `1=Пн`, `2=Вт`, `3=Ср`, `4=Чт`, `5=Пт`, `6=Сб`
- Время начала (например, `15:30`)
- Длительность в минутах (обычно `90`)

Сохранённые слоты используются в **календаре** (`/api/calendar`) для развёртки в конкретные даты диапазона.

### 4.6 Состав группы

Внутри карточки группы → вкладка «Состав».

- «Добавить ученика» → выберите из списка → опционально укажите `since` (с какой даты) и `until` (до).
- Один ученик может быть в нескольких группах одновременно.

### 4.7 Карточки клиентов CRM

`Админка → CRM → Карточки клиентов`

Карточка содержит расширенные данные ученика:

| Поле | Описание |
| --- | --- |
| ФИО (`fullName`) | Полное имя |
| Дата рождения (`birthDate`) | YYYY-MM-DD |
| Пол (`gender`) | `m` / `f` |
| Филиал (`branchId`) | К какому филиалу прикреплён |
| Тариф (`tariffId`) | Действующий абонемент |
| Дата выдачи абонемента (`subscriptionIssuedAt`) | Unix ms |
| Остаток занятий (`visitsLeft`) | Уменьшается при отметке `present`/`late` |
| Статус (`status`) | `active` / `frozen` / `inactive` |
| Менеджер (`responsibleManagerId`) | Ответственный за клиента |
| Имя родителя (`parentName`) | |
| Телефон родителя (`parentPhone`) | Виден только админу |
| Документ (`documentId`) | ИИН/паспорт; только админ |
| Комментарий (`comment`) | Заметки |
| Согласие на видео (`videoConsent`) | + дата |

Учитель видит карточки только своих учеников и **без** чувствительных полей (`documentId`, `parentPhone` маскируется).

### 4.8 Календарь занятий

`Админка → Учебный процесс → Календарь`

API: `GET /api/calendar?from=YYYY-MM-DD&to=YYYY-MM-DD&branch_id=...`

Возвращает массив событий, развёрнутых из расписания групп + проведённые внеплановые занятия.

Пример события:
```json
{
  "date": "2026-06-18",
  "weekday": 4,
  "startTime": "11:30",
  "durationMin": 90,
  "groupId": "grp_blockly_2",
  "groupName": "Blockly — Юниоры (6–8)",
  "branchId": "br_zhoshy",
  "branchName": "KURSOR — Жошы Хан",
  "courseTitle": "Алгоритм шагов",
  "teacherName": "Айбек Серикович",
  "sessionId": null,
  "conducted": false,
  "presentCount": 0,
  "topic": ""
}
```

- Если занятие уже проведено — `sessionId` заполнен, `conducted: true`, `topic` показывает тему, `presentCount` — кол-во посетивших.
- Если в расписании ничего нет, но есть внеплановое занятие в эту дату — оно тоже попадает в события с `adhoc: true`.

### 4.9 Журнал и посещаемость

`Админка → Учебный процесс → Журнал`

**Создать занятие:**
1. Выберите группу → «Новое занятие».
2. Укажите дату и тему.
3. Сохраните.

**Отметить посещаемость:**
1. Откройте занятие → таблица учеников группы.
2. Для каждого ученика выберите: `present` (был) / `late` (опоздал) / `excused` (уважительная причина) / `absent` (отсутствовал).
3. Сохранить.

**Логика списания абонемента:**
- `present` или `late` → списывается 1 занятие (если статус CRM = `active`).
- Исправление с `present`/`late` на `excused`/`absent` → возвращает 1 занятие.
- `excused` и `absent` занятие НЕ списывают.

**Автоматическая проверка отчётов:**
После сохранения посещаемости система проверяет — у всех ли присутствовавших есть загруженный отчёт (артефакт). Если нет — учителю и всем админам приходит уведомление «не загрузил отчёт по занятию ... для: Имя1, Имя2». После загрузки отчётов уведомление автоматически снимается.

### 4.10 Домашние задания

Внутри занятия → вкладка «Домашнее задание».

- Привяжите модуль (`moduleId`) и/или конкретные задачи (`taskIds`).
- Срок сдачи (`dueDate`).
- По умолчанию ДЗ назначается **всем** ученикам группы, либо можно выбрать конкретных (`studentIds`).
- После сохранения каждый назначенный ученик получает уведомление «Назначено новое домашнее задание».

Ученик видит свои ДЗ в дашборде, а также через `/api/homework/me` с прогрессом по каждой задаче.

### 4.11 Отчёты и работы детей

Внутри занятия → вкладка «Отчёты».

Поддерживаемые типы (`session_artifacts.type`):

| Тип | Назначение | Хранение |
| --- | --- | --- |
| `video` | Видеозапись занятия | **30 дней**, потом автоудаление |
| `screenshot` | Скриншот работы | Бессрочно |
| `file` | Любой файл (PDF, ZIP, и т.д.) | Бессрочно |
| `link` | Ссылка на внешний проект (Scratch, Replit) | Бессрочно |

**Загрузить:**
- Для `video` / `screenshot` / `file` — base64 dataUrl (до 50 МБ) или внешняя ссылка.
- Для `link` — URL.

Учитель должен иметь право `upload_artifacts`. Видеозапись требует, чтобы у ученика стояло согласие `videoConsent = 1` в CRM-карточке.

### 4.12 Уведомления

`Админка → Обзор → Уведомления`

API: `GET /api/notifications` — `{ items: [...], unread: N }`

Типы уведомлений:

| `type` | Когда генерируется |
| --- | --- |
| `homework` | Назначено новое ДЗ |
| `subscription` | Низкий остаток абонемента |
| `missing_report` | Не загружен отчёт по занятию (учителю и админу, идемпотентно) |
| `course_access_expiring` | Доступ учителя к курсу скоро истечёт |
| `system` | Системные сообщения |

В поле `channel` пока всегда `in_app`, но заложено под будущую интеграцию с WhatsApp/Email.

### 4.13 Импорт-экспорт

Встроены в соответствующие вкладки (отдельной страницы нет):

| Что | API | Формат |
| --- | --- | --- |
| Экспорт учеников | `GET /api/export/students` | CSV |
| Экспорт модулей | `GET /api/export/modules` | JSON |
| Экспорт материалов | `GET /api/export/materials` | JSON |
| Импорт учеников | `POST /api/import/students` | CSV |
| Импорт модулей | `POST /api/import/modules` | JSON |

> Чувствительные поля (документ, телефон родителя, дата рождения) включаются в экспорт **только** для администратора.

## 5. Кабинет учителя

URL: `/pages/teacher.html`

**Что доступно:**
- Список моих групп (где `teacher_id = я` или `assistant_id = я`)
- Внутри группы: журнал занятий, посещаемость (с правом `conduct_lessons`), ДЗ, фидбек (с `write_feedback`), отчёты (с `upload_artifacts`)
- CRM-карточки только своих учеников, **без** чувствительных данных
- Создание/редактирование модулей и задач (с правом `manage_tasks`)
- Личный календарь занятий

**Чего НЕ может:**
- Видеть чужих учеников и чужие группы
- Менять расписание, состав группы, тарифы, филиалы (это только админ)
- Видеть `documentId` и полный `parentPhone` в CRM-карточках

## 6. Кабинет ученика

URL: `/pages/dashboard.html`

**Дашборд:**
- Текущий прогресс (очки, серия)
- Бейджи
- Назначенные ДЗ
- Уведомления

**Каталог** (`/pages/catalog.html`): все доступные модули, сгруппированные по языкам.

**Задача** (`/pages/task.html?id=...`): интерфейс решения задачи.
- При первой попытке создаётся запись в `task_progress` со статусом `progress`.
- После правильного ответа — `done`, начисляются очки (на сервере) + 5 бонусных, если без подсказки.
- При первой решённой задаче выдаётся бейдж `first_code`.
- За 3/7/30 дней подряд — бейджи `streak_3`/`streak_7`/`streak_30`.

**Урок** (`/pages/lesson.html?moduleId=...`): теория (intro-экраны) + мини-задача (`miniTask`).

**Профиль** (`/pages/profile.html`): аватарка, смена пароля, мои языки и группа.

**Рейтинг** (`/pages/leaderboard.html`): все ученики, отсортированные по очкам.

## 7. Кабинет родителя

URL: `/pages/parent.html`

Родитель видит **только** данные своих детей (привязка через `parent_children`). Все запросы идут через `/api/parent/*` (read-only).

**Возможности:**

| Раздел | API |
| --- | --- |
| Список детей с тарифом, остатком занятий, группой | `GET /api/parent/children` |
| Прогресс ребёнка (очки, серия, бейджи, решённые задачи) | `GET /api/parent/progress/:studentId` |
| Посещаемость ребёнка (история занятий с темами) | `GET /api/parent/attendance/:studentId` |
| Работы и отчёты ребёнка (без внутренних видео) | `GET /api/parent/artifacts/:studentId` |
| Фидбек учителя (только не `is_internal = 1`) | `GET /api/parent/feedback/:studentId` |
| Уведомления | `GET /api/notifications` |

**Чего родитель НЕ видит:**
- Чужих детей
- Внутренних заметок учителя (`is_internal = 1`)
- Документов / IIN своего ребёнка
- Чувствительных данных других пользователей

### Как протестировать кабинет родителя

1. Запустите засев: `node server/seed-test-data.js`
2. Войдите как `parent_lesovaigerim0 / parent123` — у этого родителя 2 ребёнка.
3. На странице `/pages/parent.html` вы увидите:
   - Карточки обоих детей с фото, группой, остатком занятий, тарифом
   - Историю посещаемости (последние 4 недели)
   - Прогресс по задачам и бейджи
   - Загруженные работы (скриншоты/ссылки)
   - Отзывы учителей (если есть)
   - Уведомление о низком остатке абонемента

## 8. REST API — примеры запросов

Все примеры используют `curl`. JWT хранится в переменной `$TOKEN`.

### Логин

```bash
TOKEN=$(curl -s -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"login":"admin","password":"admin"}' \
  | jq -r '.token')
```

### Создать филиал

```bash
curl -X POST http://localhost:3000/api/branches \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"KURSOR — Ташенова","address":"ул. Ташенова, 8"}'
```

### Создать тариф

```bash
curl -X POST http://localhost:3000/api/tariffs \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"Стандарт","visitsCount":8,"durationDays":30,"price":30000,"comment":"2 занятия в неделю"}'
```

### Создать группу

```bash
curl -X POST http://localhost:3000/api/groups \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name":"Python — Старт (10–12)",
    "branchId":"br_zhoshy",
    "teacherId":"t_dina",
    "courseId":"python_1",
    "lessonKind":"main"
  }'
```

### Добавить слот расписания (Понедельник 15:30, 90 мин)

```bash
curl -X POST http://localhost:3000/api/groups/grp_python_5/schedule \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"weekday":1,"startTime":"15:30","durationMin":90}'
```

### Добавить ученика в группу

```bash
curl -X POST http://localhost:3000/api/groups/grp_python_5/members \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
```

## 9. Решение типичных проблем

### Сервер не запускается / порт занят

Найти процесс на 3000:
```bash
lsof -i :3000
# или
pkill -f "node server/index.js"
```

### Забыл пароль admin

Сбросьте через SQL:
```bash
node -e "
const bcrypt = require('bcryptjs');
const db = require('./server/db');
db.prepare('UPDATE users SET password_hash=? WHERE login=?')
  .run(bcrypt.hashSync('admin', 10), 'admin');
console.log('Пароль admin сброшен на admin');
"
```

### Хочу полностью сбросить базу

```bash
rm server/db/kursor.sqlite*
npm start          # пересоздаст БД и засеет контент
node server/seed-test-data.js  # засеет тестовые данные
```

### Сидинг падает: «table x has no column named ...»

Это значит, что миграции не применились. Возможные причины:

1. Старая `*.sqlite` от предыдущей версии — удалите её и запустите заново.
2. Запустите `node server/db.js` отдельно — миграции применятся.
3. В крайнем случае установите в `.env`: `RUN_DB_FIX=true` — один раз запустится `fix_db.js`.

### WebSocket не подключается

Проверьте, что `token` корректно передан в URL: `ws://host:3000/ws?token=<jwt>`.

В nginx — обязательны заголовки `Upgrade` и `Connection "upgrade"` (см. README).

### Кабинет родителя пустой

1. Проверьте, что у пользователя роль `parent`.
2. Проверьте, что есть привязка `parent_children`:
   ```sql
   SELECT * FROM parent_children WHERE parent_id = 'par_xxx';
   ```
3. Если пусто — привяжите детей через админку или `PUT /api/users/:parentId/children`.

### Артефакты не отображаются в кабинете родителя

Проверьте:
- У занятия (`lesson_session`) есть `group_id`, и ребёнок состоит в этой группе.
- У артефакта `deleted = 0` (для видео срок не истёк).
- Тип артефакта корректный (`screenshot`, `file`, `link`, `video`).

## 🆘 Если что-то не работает

1. Откройте `http://localhost:3000/api/health` — должно вернуть `{ status: "ok", users: N, tasks: N, modules: N }`.
2. Проверьте консоль сервера — там видны все API-запросы и ошибки.
3. Проверьте консоль браузера (F12) — там видны ошибки фронтенда.
4. Сбросьте БД (см. выше) и пере-засейте тестовые данные.

Все исходники открыты — изучайте `server/routes-*.js` для понимания логики.
Удачи! 🚀
