# Внесённые исправления (KURSOR CRM)

Дата: исправления по запросу — баг «Invalid Date», загрузка файлов в материалы,
работы/видео по каждому ученику в окне занятия, перевод на казахский.

## 1. Баг «Invalid Date»
Причина: даты сохранялись как число `getTime()` в колонку типа `TEXT`. SQLite
хранил их строкой ("1781568000000"), а `new Date("1781568000000")` → Invalid Date
(числовая строка не парсится как timestamp).

Что сделано:
- `public/js/app.js` — добавлен безопасный парсер `toDate/fmtDate/fmtDateTime`
  (понимает число-мс, числовую строку и ISO-строку). Экспортирован глобально.
- `public/admin/index.html` — локальные `fmtDate/fmtDateTime` сделаны такими же
  безопасными.
- `public/pages/parent.html, dashboard.html, profile.html, teacher.html` — все
  прямые вызовы `new Date(...).toLocaleDateString()` заменены на `fmtDate(...)`.
- Миграция данных НЕ нужна: парсер корректно читает уже сохранённые записи.

## 2. Загрузка файлов в «Материалы» (PDF / PPTX / XLSX / DOCX)
- `server/index.js` — для `/api/materials` поднят лимит тела до 70 МБ.
- `server/routes-materials.js` — POST/PUT принимают `dataUrl` (base64) + `fileName`,
  сохраняют файл в `public/uploads/materials/<id>.<ext>` и кладут URL в `content`.
  Лимит 50 МБ, белый список расширений (pdf, ppt(x), xls(x), doc(x), png, jpg, …).
- `public/admin/index.html` — в форме материала добавлен выбор файла; файл читается
  как base64 и отправляется на сервер.

## 3. Работы и видеоотчёты по каждому ученику в окне проведения занятия
Бэкенд уже поддерживал это (`session_artifacts`, `/api/session-artifacts`,
видео авто-удаляется через 30 дней; видео требует согласия `video_consent`).
Добавлен фронтенд:
- `public/admin/index.html` — окно «Посещаемость» переделано в «Проведение занятия».
  У каждого ученика: статус + кнопки «📎 Работа» (картинка/файл + текстовое описание)
  и «🎬 Видео», список загруженного с удалением. Всё это видно родителю в его кабинете.

## 4. Перевод на казахский
- `public/js/i18n.js` — добавлены ключи для нового окна занятия, формы материалов и
  формы ДЗ (ru + kk).
- Жёстко зашитые русские подписи в этих формах заменены на `I18N.t(...)`.

> Примечание: полный обход всех ~1500 строк интерфейса на остаточные русские строки
> и визуальный редизайн под alfaCRM (календарь-неделя) — следующий этап.
